# /review-content

生成されたコンテンツ（キャプション・画像プロンプト）を人間レビュー用にまとめて表示する。
将来の自動化フローにおける「人間チェックポイント」として機能する。

## 引数
`$ARGUMENTS` — レビューしたいコンテンツのJSON文字列またはファイルパス

## 実行手順

1. 引数からコンテンツを読み込む

2. 以下の観点でチェックリストを生成する

```
## コンテンツレビュー — [モード] [日付]

### Instagram
[ ] キャプションの英語は自然か？
[ ] 日本語説明は商品内容と合っているか？
[ ] ハッシュタグは5個以内か？
[ ] #PRが必要なのに抜けていないか？

### TikTok
[ ] 日本語本文は自然か？
[ ] 英語一言は流れに合っているか？
[ ] #PRが含まれているか（通常modeの場合）

### YouTube
[ ] タイトルは70文字以内か？
[ ] 最初の40文字にキーワードが入っているか？
[ ] 固定フッターが含まれているか？

### 画像プロンプト
[ ] baby_cuboスタイルが指示されているか？
[ ] AIっぽさを排除する指示があるか？
[ ] 帽子の指示があるか？

## 承認
- [ ] 承認して投稿 → app.py を開いて投稿タブへ
- [ ] 修正が必要 → [修正内容を記載]
```

3. 承認された場合は投稿手順を案内する

## 用途
- 自動生成コンテンツの人間チェック
- Phase 3自動化の中間レビューポイント
- 品質管理ログの作成
